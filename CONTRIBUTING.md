# Contributing

Thanks for taking the time to help improve this blog! Contributions are welcome, especially corrections to blog posts.

## Fixing a typo or mistake in a blog post

Every blog post has an **"Edit this page on GitHub"** link at the bottom. Clicking it takes you directly to the source file on GitHub where you can propose a correction.

1. Click **"Edit this page on GitHub"** at the bottom of the post
2. Make your change in the GitHub editor
3. Submit a pull request with a brief description of what you fixed

## Reporting an error

If you'd rather not open a pull request, use the **"Report a mistake"** link at the bottom of any post to open a pre-filled GitHub issue.

## Running the project locally

```bash
# Install dependencies
pnpm install

# Start dev server
pnpm dev

# Build for production
pnpm build
```

## Guidelines

- Keep changes focused — one fix per pull request
- Don't change the author's voice or meaning, only correct factual errors or typos
- Code changes unrelated to blog content should be discussed in an issue first
